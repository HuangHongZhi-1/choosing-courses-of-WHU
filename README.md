# 🎓 武汉大学教务系统 - 智能选课系统

一个功能完整、安全可靠、智能强大的选课工具。支持实时输入、同时管理多个课程、自动保存和一键恢复。

## ✨ 核心特性

- ✅ **实时输入课程信息** - 课程号、课程名、教师名直接输入，无需预先配置
- ✅ **完整课程管理** - 增删改查，完全掌控
- ✅ **实时监控课程** - 每5秒检查一次课程状态，发现有空位立即选课
- ✅ **自动化选课流程** - 自动登录、处理验证码、逐个选课
- ✅ **数据完整保存** - 自动保存课程计划和选课历史
- ✅ **一键恢复功能** - 即使程序关闭，也能完全恢复所有课程信息
- ✅ **企业级安全** - 凭证不存储在任何文件，密码被隐藏

## 🚀 快速开始（30秒）

```bash
python grab_whu_interactive.py
```

然后按照菜单提示操作即可！

## 📋 三个脚本

| 脚本 | 用途 | 命令 |
|------|------|------|
| **grab_whu_interactive.py** ⭐ | 交互式智能选课系统（最推荐） | `python grab_whu_interactive.py` |
| grab_whu_courses.py | 配置化自动选课 | `python grab_whu_courses.py` |
| whu_login.py | 登录测试和课程查询 | `python whu_login.py` |

### grab_whu_interactive.py 的8个菜单功能

```
1. ➕ 添加课程              - 实时输入课程号、课程名、教师名
2. 📋 查看所有课程          - 显示所有已添加的课程
3. ❌ 删除课程              - 删除不需要的课程
4. ✏️ 修改课程              - 修改课程信息
5. 👁️ 实时监控课程状态      - 每5秒检查一次课程状态
6. 🚀 开始选课              - 自动登录并选课
7. 📊 查看选课历史          - 查看最近10次的选课记录
8. ♻️ 恢复上次选课计划      - 一键恢复已删除的课程
0. 🚪 退出                 - 退出程序
```

## 详细配置说明

### config.json 参数

| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `system_type` | string | ✓ | 教务系统类型: `jw_system`, `xuexi_tong`, `moodle`, `general` |
| `base_url` | string | ✓ | 教务系统主页地址 |
| `username` | string | ✓ | 学号或用户名 |
| `password` | string | ✓ | 登录密码 |
| `grab_time` | string | ✓ | 抢课时间 (格式: `YYYY-MM-DD HH:MM:SS`) |
| `courses` | array | ✓ | 要抢的课程列表 |
| `max_retries` | int | ✗ | 最大重试次数 (默认: 3) |
| `timeout` | int | ✗ | 请求超时时间(秒) (默认: 10) |

### courses 数组格式

```json
{
  "id": "course_001",
  "name": "课程名称",
  "term": "2026-1"
}
```

## 常见问题

### Q: 如何找到课程ID?

**A:** 在教务系统中:
1. 进入选课界面
2. 按F12打开开发者工具
3. 找到选课按钮的属性或API请求
4. 复制课程ID

### Q: 验证码怎么处理?

**A:** 
- **JW系统**: 脚本会自动处理大部分验证码
- **需手动输入**: 在脚本运行时，系统会提示你输入验证码

### Q: 为什么登录失败?

**A:** 常见原因:
1. 账号密码错误
2. 系统维护中
3. IP被限制
4. 密码格式需要特殊处理

尝试以下步骤:
1. 检查 `grab_website.py` 日志
2. 手动登录验证账号密码
3. 修改 `website_adapter.py` 适配你的系统

### Q: 如何提高抢课成功率?

**A:** 
1. ✅ 设置正确的 `grab_time`
2. ✅ 增加 `max_workers` 并发数
3. ✅ 减少 `timeout` 值(但不要太小)
4. ✅ 提前测试登录流程
5. ✅ 确保网络稳定

### Q: 可以同时抢多个课程吗?

**A:** 可以！添加多个课程到 `courses` 数组:

```json
"courses": [
  {"id": "001", "name": "课程1"},
  {"id": "002", "name": "课程2"},
  {"id": "003", "name": "课程3"}
]
```

脚本会并行抢取所有课程。

## 自定义和扩展

### 添加新的教务系统

1. 创建新的适配器类:

```python
class YourSystemAdapter(WebsiteGrabberAdapter):
    def login(self) -> bool:
        # 实现登录逻辑
        pass
    
    def grab_course(self, course_info: Dict) -> bool:
        # 实现抢课逻辑
        pass
```

2. 在 `get_adapter()` 函数中注册:

```python
adapters = {
    'your_system': YourSystemAdapter,
    # ...其他系统
}
```

3. 在配置中使用:

```json
{
  "system_type": "your_system",
  ...
}
```

### 修改抢课策略

编辑 `WebsiteCoursGrabber.grab_single_course()` 方法来实现自定义策略。
